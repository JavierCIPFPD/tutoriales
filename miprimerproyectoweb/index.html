Base del proyecto

Carpetas y archivos minimos recomendados

Con esta estructura puedes mantener orden, separar responsabilidades y subir el proyecto a un hosting estatico sin depender de herramientas complejas.

Arbol sugerido

miprimerproyectoweb/
|-- index.html
|-- assets/
|   |-- css/
|   |   `-- style.css
|   |-- js/
|   |   `-- main.js
|   `-- img/
|       `-- favicon.svg

Que aporta cada parte

  • index.html contiene la estructura y el contenido visible.
  • assets/css guarda el diseño y ayuda a mantener limpio el HTML.
  • assets/js concentra la logica interactiva.
  • assets/img separa los recursos visuales.
  • favicon mejora la presentacion del sitio.

Funcion de cada archivo

Que debe contener un proyecto pequeno

HTML

El archivo index.html define el contenido, el orden de lectura y los enlaces a estilos, scripts e iconos.

Es la pieza imprescindible: si existe este archivo y las rutas son correctas, el hosting suele poder servir la web sin configuracion extra.

CSS

El archivo style.css se ocupa del aspecto visual, la adaptacion a movil y efectos como el parallax.

Separar estilos del HTML hace que el proyecto sea mas mantenible y profesional.

JavaScript

El archivo main.js permite cambiar entre tema claro y oscuro y activar pequenas mejoras de experiencia.

Si el proyecto no necesita interaccion, puede ser muy pequeno, pero sigue siendo util dejarlo preparado.

Detalles que marcan la diferencia

Lineas concretas que conviene entender

Estas lineas parecen pequenas, pero mejoran compatibilidad, accesibilidad, posicionamiento y experiencia de uso.

Cabecera del HTML

<!doctype html>
<html lang="es">
<meta charset="utf-8">
<meta name="viewport" content="width=device-width, initial-scale=1">
<meta name="description" content="Resumen claro del sitio">
<title>Nombre de la pagina</title>
<link rel="icon" href="assets/img/favicon.svg" type="image/svg+xml">
  • doctype: activa el modo estandar del navegador.
  • lang="es": ayuda a lectores de pantalla y buscadores.
  • charset="utf-8": evita problemas con caracteres especiales.
  • meta viewport: hace que la web sea responsive en movil.
  • meta description: mejora la presentacion del sitio en resultados de busqueda.
  • title: identifica claramente la pagina.

Conexion de estilos y scripts

<link rel="stylesheet" href="assets/css/style.css">
...
<script src="assets/js/main.js" defer></script>
  • rel="stylesheet": carga el CSS del proyecto.
  • href: apunta al archivo correcto dentro de la estructura.
  • defer: evita bloquear la carga visual de la pagina.

Claves del CSS

:root {
  --bg: #eef6f8;
  --text: #12202d;
}

@media (max-width: 720px) {
  .grid--three {
    grid-template-columns: 1fr;
  }
}
  • variables CSS: facilitan cambiar tema y colores.
  • @media: adapta la composicion al movil.
  • 1fr: hace que una columna ocupe todo el ancho disponible.

Claves del JavaScript

const root = document.documentElement;
const savedTheme = localStorage.getItem('preferred-theme');

themeToggle?.addEventListener('click', () => {
  const nextTheme = root.dataset.theme === 'dark' ? 'light' : 'dark';
  setTheme(nextTheme);
});
  • document.documentElement: permite cambiar atributos globales del sitio.
  • localStorage: recuerda el tema elegido por la persona usuaria.
  • dataset.theme: guarda el tema activo en el HTML para que el CSS lo detecte.
  • addEventListener: conecta el boton con el cambio visual.
